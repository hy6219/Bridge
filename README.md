## 03. 커밋 메시지 규칙

- <타입>: <제목> (제목은 최대 50 글자까지 입력)
- <타입> 리스트

1.  feat    : 기능 (새로운 기능)
2.  fix     : 버그 (버그 수정)
3.  refactor: 리팩토링
4.  style   : 스타일 (코드 형식, 세미콜론 추가: 비즈니스 로직에 변경 없음)
5.  docs    : 문서 (문서 추가, 수정, 삭제)
6.  test    : 테스트 (테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없음)
7.  chore   : 기타 변경사항 (빌드 스크립트 수정 등)
